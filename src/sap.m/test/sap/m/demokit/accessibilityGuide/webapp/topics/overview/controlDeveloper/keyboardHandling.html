Keyboard Usage of ARIA Role Mapped Controls

Overview

People who interact with a page using a mouse can access any visible element on this page directly by clicking on it. To interact with an element, keyboard users need to move the focus sequentially through all interactive elements on a page until they reach the desired element.


Navigating through a webpage with a keyboard is possible using the TAB key to move forward and the SHIFT+TAB combination to move backwards. These keys move the focus only between elements of the page that are focusable. If an element can be focused using the keyboard, then it should be interactive. That is, the user should be able to do something with it and produce a change in the state of the application (for example, activating a link or changing an option).

Tabindex

HTML elements like buttons, links, inputs and others, can receive focus.


<button>Press Tab</button>

Some HTML elements do not receive focus by default. To make such elements focusable, you can use the tabindex global attribute. This attribute indicates that the element can be focused, and where it participates in the sequential keyboard navigation (usually with the Tab key, hence the name). An example for usage of the tabindex is when a custom widget with role combobox needs to be created. 

<div role="combobox" tabindex="0" ...> ... </div>

When using tabindex with value -1 you can prevent an interactive element to be focused. This might be used in specific scenarios. For example, it is used in the design of the Input control, where the icons container has a tabindex -1 as there are other keyboard means how to activate the functionally with keyboard and we want to optimize the tab chain.
