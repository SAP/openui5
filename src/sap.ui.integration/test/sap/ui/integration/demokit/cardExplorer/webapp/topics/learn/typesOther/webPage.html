WebPage Card

The WebPage Card allows to embed HTML page in the content area, which will be isolated in iframe.

Properties

Property Type Default Value Required Description Schema Version Since
src string Yes The URL of the web page to be embedded. 1.33.0 1.90
sandbox string (no value, applies all restrictions) No Determines what restrictions are applied to the content of the embedded page. By default full restrictions are applied. This is the preferred value if the page origin is not trusted. Set this to space-separated tokens to lift particular restrictions. For more detailed documentation and possible tokens read the iframe sandbox attribute documentation.
Note: This property is experimental. 		1.33.0 1.90
omitSandbox boolean false No If set to "true" the "sandbox" attribute will not be added to the iframe. This will remove the default restrictions from the embedded page, which opens a security vulnerability! Do this only if the content of the embedded page is fully trusted.
Note: Omitting the sandbox attribute opens a security vulnerability and must be done with great caution and only if the content of the embedded page is fully trusted.
Note: This property is experimental. 		1.65.0 1.126
minHeight sap.ui.core.CSSSize 150px No Defines the minimum height of the content. 1.33.0 1.90
allow string No Determines what permissions-policy features are available to the content of the embedded page. No features are added by default. This is the recommended behavior when the page origin is not trusted. For more detailed documentation read about iframe property 'allow' and for list of options see Permissions Policy.
Note: The option "fullscreen" is not supported on all browsers, consider using the option "allowFullscreen" instead. For specific browser compatibility information, refer to allow property Permissions Policy.
Note: This property is experimental. 		1.65.0 1.126
allowFullscreen boolean false No Determines if the web page can activate fullscreen mode by calling requestFullscreen() method. For more information, read the iframe allowFullscreen attribute documentation.
Note: This property is experimental. 		1.65.0 1.126

Disclaimer: If you change the default values of any of the allow, allowFullscreen, sandbox, or omitSandbox properties, you're responsible for ensuring the secure behavior of the web page.

Example

Define the manifest.json of the card:

{
	"sap.app": {
		"id": "card.explorer.webpage.card",
		"type": "card",
		"title": "Sample of a WebPage Card",
		"subTitle" :"Sample of a WebPage Card",
		"applicationVersion": {
			"version": "1.0.0"
		},
		"shortTitle": "A short title for this Card",
		"info": "Additional information about this Card",
		"description": "A long description for this Card",
		"tags": {
			"keywords": [
				"WebPage",
				"Card",
				"Sample"
			]
		}
	},
	"sap.ui": {
		"technology": "UI5"
	},
	"sap.card": {
		"type": "WebPage",
		"configuration": {
			"parameters": {
				"minHeight": {
					"description": "Minimum height of the content",
					"type": "string",
					"value": "25rem"
				}
			}
		},
		"header": {
			"title": "OpenUI5 Events",
			"icon": {
				"src": "sap-icon://desktop-mobile"
			}
		},
		"content": {
			"src": "https://openui5.org/events",
			"minHeight": "{parameters>/minHeight/value}"
		}
	}
}

Create the view to display the card:

<mvc:View xmlns:w="sap.ui.integration.widgets">
	<w:Card manifest="./manifest.json" width="400px" height="auto" />
</mvc:View>
