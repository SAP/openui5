Overview

There are cases where a manifest.json is not enough to render a card. Sometimes the card needs translation files or other static resources, such as, images and icons. In such cases, you can place the manifest.json in a separate folder and add all required static resources there. The paths to those static resources will be relative to the manifest.json.

A card with manifest.json and static resources is a card bundle.

An example folder structure of a card bundle which includes a manifest.json, i18n translation files and an icon:

The manifest.json of the example card bundle: 

{
	"sap.app": {
		"type": "card",
		"i18n": "i18n/i18n.properties",
		"id": "my.card"
	},
	"sap.card": {
		"type": "List",
		"header": {
			"title": "{{appTitle}}",
			"subTitle": "{{appDescription}}",
			"icon": {
				"src": "./icons/edit.png"
			}
		},
		"content": {
			...
		}
	}
}

NOTE: The paths to the i18n translation files and to the header icon are relative to the manifest.json
