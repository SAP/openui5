Overview

Cards integration into a host environment - applications, dashboards, or any HTML page? Although cards are acting as self-contained UI elements, there are integration capabilities and requirements, that should be enabled and handled by the host environment. Examples: passing parameters, handling events, etc., which we are covering in details in the next sections.

Responsibilities of the host environment

Depending on the cards content, it is necessary for the host environment to provide additional functionality such as:

  • Ensuring that the backend data required by the card is accessible.
  • Passing global parameters.
  • Handling of actions and events, that occur on the cards content during user interactions.
  • Providing the layout, that the card is used in.

OpenUI5 vs SAPUI5

Depending on the UI5 delivery you are using (SAPUI5 or OpenUI5) there are different card types:

OpenUI5

  • List Card
  • Table Card
  • Object Card
  • Component Card
  • Calendar Card (Experimental)

SAPUI5 - adds Timeline and Analytical

  • List Card
  • Table Card
  • Object Card
  • Component Card
  • Timeline Card
  • Analytical Card
