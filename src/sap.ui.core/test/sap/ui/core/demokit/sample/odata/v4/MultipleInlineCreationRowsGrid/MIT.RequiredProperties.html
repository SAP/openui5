Manual Integration Test: Required Properties

DESCRIPTION

Test of the Multiple Inline Creation Rows Grid application with inline creation rows, required properties, and resetting changes.

PREPARATION

To check console errors open the developer tools before starting the application and check with each step, that there are no errors and warnings.

Testing with mock data

Start the Multiple Inline Creation Rows Grid application with following URL (adjust host and port if necessary): http://<host>:<port>/test-resources/sap/ui/core/demokit/sample/common/index.html?component=odata.v4.MultipleInlineCreationRowsGrid

REALIZE and CHECK

Test the app within draft mode.

  1. Check: A list with the title "3 Products" and two empty rows, followed by three products appears.
  2. Enter "Product 110" into the Name field of the first empty row and press "Enter" key.
  3. Check: A warning message "ID must not be empty" appears.
  4. Check: The ID field has a colored border indicating the warning.
  5. Click on the Reset Changes button on the "Products" table header
  6. Check: The first row is empty again and the Name Product 110 disappears.
  7. Check: The warning message "ID must not be empty" disappears.
  8. Again, enter "Product 110" into the Name field of the first empty row and press "Enter" key.
  9. Check: A warning message "ID must not be empty" appears.
  10. Check: The ID field has a colored border indicating the warning.
  11. Enter "110" into the ID field of the first empty row and press "Enter" key.
  12. Check: The warning message and the colored border of the ID field disappear.
  13. Check: The Delete product button of the row becomes active.
  14. Check: A new empty row appears.
  15. Check: The product with ID "110" now has the state "Persisted".
  16. Check: A table with title "Product 110, 0 Parts" appears. It contains two empty rows.
  17. Check: The Delete part button is inactive for the empty rows.
  18. Enter "Part 50" into the Description field of the first empty row and press "Enter" key.
  19. Check: A warning message "ID must not be empty" appears.
  20. Check: The ID field has a colored border indicating the warning.
  21. Click on the Reset Changes button on the "Parts" table header
  22. Check: The first row is empty again and the Description Part 50 disappeared.
  23. Again, enter "Part 50" into the Description field of the first empty row and press "Enter" key.
  24. Check: A warning message "ID must not be empty" appears.
  25. Check: The ID field has a colored border indicating the warning.
  26. Enter "50" into the ID field of the first empty row and press "Enter" key.
  27. Check: The warning message and the colored border of the ID field disappear.
  28. Check: The Delete part button of the row becomes active.
  29. Check: The table title changes to "Product 10, 1 Parts".
  30. Check: A new empty row appears.
  31. Enter "Part 50" into the Description field of the first empty row and press "Enter" key.
  32. Check: A warning message "ID must not be empty" appears.
  33. Check: The ID field has a colored border indicating the warning.
  34. On the product table, enter "Product 110" into the Name field of the first empty row and press "Enter" key.
  35. Check: A warning message "ID must not be empty" appears.
  36. Check: The ID field has a colored border indicating the warning.
  37. Click on the Reset button on the application header bar
  38. Check: Both tables are empty again and contain only two inactive rows. All messages disappear.
