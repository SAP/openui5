Manual Integration Test: Data Aggregation

DESCRIPTION

Test scrolling and expanding of aggregated data.

PREPARATION

To check console errors open the developer tools before starting the application and check with each step, that there are no errors and warnings. Ignore initial message bundle errors.

Testing with mock data

Start the Data Aggregation application URL (adjust host and port if necessary):
  • local: http://<host>:<port>/testsuite/test-resources/sap/ui/core/demokit/sample/common/index.html?component=odata.v4.DataAggregation

REALIZE and CHECK

Expand the third node

  1. After the application is started, click on the expand button in the third row with country Germany.
  2. Check: Two new rows appear below the third row with country Germany, region Lower-Saxony and region Saxony.

Scroll to the end of the table

  1. Click on Germany to move the focus to the table. Press CTRL-END to scroll to the end of the table.
  2. Check: The row with the country Germany and region Hessia is displayed at the top.

Expand the last visible node

  1. Click on the expand button of the last visible row with the region Baden-Württemberg.
  2. Check: The scrollbar of the table has changed.
  3. Scroll down to the end of the table.
  4. Check: The first visible row has country Germany and region Bavaria.

Expand the last visible node

  1. Click on the expand button of the last visible row with the region Baden-Württemberg and the segment Small.
  2. Check: The scrollbar of the table has changed.
  3. Click on Germany to move the focus to the table. Press CTRL-END to scroll to the end of the table.
  4. Check: The table should have the following content:
    Level Expand Country Region Segment Account Responsible Sales Amount Sales Number Subtotal

    3 Not Expanded Germany Baden-Württemberg Mid-size 526.122
