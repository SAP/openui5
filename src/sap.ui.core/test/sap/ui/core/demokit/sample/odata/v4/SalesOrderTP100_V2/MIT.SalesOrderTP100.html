Manual Integration Test: Sales Order TP100

DESCRIPTION

Test Sales Orders TP100 application.

PREPARATION

To check console errors open the developer tools before starting the application and check with each step, that there are no errors and warnings.

Testing with mock data

Start the Sales Orders application with following URL (adjust host and port if necessary):
  • local: http://localhost:8080/test-resources/sap/ui/core/demokit/sample/common/index.html?component=odata.v4.SalesOrderTP100_V2

Testing with back end

Configure your IDE to be able to execute the Sales Orders application against an ABAP back-end system.
Start the Sales Orders application with the URL (adjust host and port if necessary):
  • local: http://localhost:8080/test-resources/sap/ui/core/demokit/sample/common/index.html?component=odata.v4.SalesOrderTP100_V2&realOData=true

REALIZE and CHECK

Check paging

  1. After the application is started, click on the More button in the Sales Orders TP100 table.Note: without back end you will get only 10 entries in total

Check context

  1. Select one entry in the Sales Orders TP100 table and check that the related Sales Order Items are shown
