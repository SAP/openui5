Manual Integration Test: Create Sales Order

DESCRIPTION

Test entity creation with the Sales Orders application.

PREPARATION

To check console errors open the developer tools before starting the application and check with each step, that there are no errors and warnings.

Testing with mock data

Start the Sales Orders application with following URL (adjust host and port if necessary):
  • local: http://localhost:8080/testsuite/test-resources/sap/ui/core/demokit/sample/common/index.html?component=odata.v4.SalesOrders

Testing with back end

Configure your IDE to be able to execute the Sales Orders application against an ABAP back end system.
Start the Sales Orders application with the URL (adjust host and port if necessary):
  • local: http://localhost:8080/testsuite/test-resources/sap/ui/core/demokit/sample/common/index.html?component=odata.v4.SalesOrders&realOData=true

REALIZE and CHECK

Create, modify and delete of an unsaved sales order

  1. Check: (without back end only) The sales orders count is 10.
  2. After the application is started, select the first sales order
  3. Click on the ValueHelp button for Categoryin the Sales Order Line Items table.
  4. Check: The title of the Value Help is "Value Help: Category (Modified Mapping)".
  5. Close the Value Help dialog.
  6. Click on the Create button in the Sales Orders table.
  7. Check: The table below the Create New Sales Order dialog gets a new row with an empty Sales Order ID and the Note as displayed in the Create New Sales Order dialog.
  8. Check: That for input field Currency Code a value help button is displayed.
  9. Press the value help button and check that a value help dialog is opened.
  10. Close the Value Help dialog.
  11. Modify the content of the Note input field of the Create New Sales Order dialog.
  12. Check: The Note of the first sales order (table below the dialog) is updated automatically after leaving the input field of the dialog.
  13. Click on OK in the Create New Sales Order dialog.
  14. Check: (without back end only) The sales orders count is 11.
  15. Select the newly created sales order.
  16. Modify the content of the Note in the newly created sales order.
  17. Click on the Delete button in the Sales Orders table.
  18. Click on the Cancel button in the Sales Order Deletion dialog.
  19. Check: The newly created sales order is still visible.
  20. Click on the Delete button in the Sales Orders table.
  21. Click on the OK button in the Sales Order Deletion dialog.
  22. Check: The newly created sales order is gone.
  23. Check: (without back end only) The sales orders count is 10.

Create a sales order, save, modify again, save and delete

  1. Click on the Create button in the Sales Orders table.
  2. Modify the content of the Note input field of the Create New Sales Order dialog.
  3. Click on OK in the Create New Sales Order dialog.
  4. Check:The Confirm selected sales order button in the Sales Orders table is disabled.
  5. Click on the Save button in the Sales Orders table.
  6. Check: You get a success message for creating the sales order.
  7. Click on OK in the Success dialog.
  8. With back end only:
    1. Check:The Confirm selected sales order button in the Sales Orders table is enabled.
    2. Check: The Sales Order ID has been updated by the server.
    3. For the newly created sales order enter a text in the Note field .
    4. Click on Save button in the Sales Orders table.
  9. Select the saved sales order.
  10. Click on Delete button in the Sales Orders table.
  11. Click on the OK button in the Sales Order Deletion dialog.
  12. Check:You get a success message for the deleted sales order.
  13. Click on OK in the Success dialog.
  14. Check: The newly created sales order is gone.

Create a sales order, save and refresh the sales orders

  1. Click on the Create button in the Sales Orders table.
  2. Modify the content of the Note input field of the Create New Sales Order dialog.
  3. Click on OK in the Create New Sales Order dialog.
  4. Click on the Save button in the Sales Orders table.
  5. Check: You get a success message for creating the sales order.
  6. Click on OK in the Success dialog.
  7. Check: (without back end only) The sales orders count is 11.
  8. Click on Refresh button in the Sales Orders table.
  9. Check: The first sales order is the same as the one from the initial application start.
  10. Check (with back end only): The new sales order is inserted at the right position (considering sort and filter criterion); maybe you have to click on the More button of the Sales Orders table to see the new sales order.

Create a sales order, refresh/filter w/o saving -> expected "pending changes" message

  1. Click on the Create button in the Sales Orders table.
  2. Click on OK in the Create New Sales Order dialog.

Cancel refresh

  1. Click on Refresh button in the Sales Orders table.
  2. Check: You get a warning about pending changes.
  3. Click on the Cancel button.
  4. Click on Refresh all button.
  5. Check: You get a warning about pending changes.
  6. Click on the Cancel button.
  7. With back end only:
    1. Enter a numerical value in the Filter by gross amount... search field above the Sales Orders table and press Enter.
    2. Check: You get a error because of unsaved changes.
    3. Click on the Close button.

Confirm refresh

  1. Click on Refresh or Refresh all button.
  2. Check: You get a warning about pending changes.
  3. Click on the Ok button.
  4. Check: The newly created sales order is gone.

Create a sales order, press "cancel sales order changes" w/o saving

  1. Click on the Create button in the Sales Orders table.
  2. Click on OK in the Create New Sales Order dialog.
  3. Click on Cancel sales orders changes button in the Sales Orders table.
  4. Check: The newly created sales order is gone.
  5. Check: (without back end only) The sales orders count is 10.

Cancel or resume a failed creation of a sales order

With back end only:
  1. Click on the Create button in the Sales Orders table.
  2. Click on OK in the Create New Sales Order dialog.
  3. Type in RAISE_ERROR into the Note input field of the new row.
  4. Click on the Save button in the Sales Orders table.
  5. Check:You get an error regarding invalid value for the Note.
  6. Click on Refresh button in the Sales Orders table.
  7. Check: You get a warning about pending changes.
  8. Click on the Cancel button.
  9. Click on Cancel sales orders changes button in the Sales Orders table.
  10. Check: The newly created sales order is gone.
  11. Click on the Create button in the Sales Orders table.
  12. Type in RAISE_ERROR into the Note input field of the Create New Sales Order dialog.
  13. Click on OK in the Create New Sales Order dialog.
  14. Click on the Save button in the Sales Orders table.
  15. Check: A POST request for the created entity is sent to the server and you got the above error.
  16. Click again on the Save button in the Sales Orders table.
  17. Check: A POST request for the created entity is sent to the server and you got the above error again.
  18. Modify the content of the Note in the newly created sales order.
  19. Click on the Save button in the Sales Orders table.
  20. Check: You get a success message for creating the sales order.
  21. Click on OK in the Success dialog.
  22. Check: The Sales Order ID has been updated by the server
